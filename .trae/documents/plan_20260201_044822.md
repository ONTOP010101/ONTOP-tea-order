## 问题分析
用户使用的是GP-U80250 Series打印机，通过USB连接到电脑。打印机可以正常打印测试页，但是：
1. 使用 `print` 命令时出现 "无法初始化设备 PRN" 的错误
2. 使用 PowerShell 命令 `Get-Process | Out-String | Out-Printer -Name "GP-U80250 Series"` 可以正常出纸

## 解决方案
修改打印机服务代码，使用 PowerShell 命令来打印订单，而不是使用 `print` 命令。

## 具体修改步骤

1. **修改打印机服务代码** (`printer.service.ts`)
   - 添加使用 PowerShell 命令打印的功能
   - 优先使用 PowerShell 命令打印
   - 添加详细的错误处理和日志输出
   - 确保打印机名称正确设置为 "GP-U80250 Series"

2. **测试打印机连接**
   - 重启后端服务
   - 测试下单流程，验证打印功能是否正常
   - 检查打印机连接状态和打印队列

3. **优化打印命令**
   - 确保 PowerShell 命令正确执行
   - 确保打印内容正确发送到打印机
   - 添加重试机制，确保打印任务能够成功执行

## 技术实现
- 使用 Node.js 的 child_process 模块执行 PowerShell 命令
- 使用 Out-Printer 命令打印订单内容
- 添加详细的日志输出，方便调试
- 确保即使打印失败，订单也能正常保存和处理

## 预期效果
- 打印机能够正确接收和打印订单
- 后端能够正确处理打印结果和错误
- 即使打印机连接失败，订单也能正常保存和处理
