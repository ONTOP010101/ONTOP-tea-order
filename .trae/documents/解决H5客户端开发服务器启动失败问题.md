## 问题分析
从错误信息来看，主要遇到以下问题：
- **端口冲突**：h5-client的5174端口可能被占用
- **进程冲突**：多个开发服务器进程同时运行导致冲突
- **依赖问题**：可能存在依赖安装不完整的情况
- **错误代码3221225786**：通常表示进程被强制终止

## 解决方案

### 步骤1：检查并释放端口
```bash
# 检查5174端口（h5-client）
netstat -ano | findstr :5174

# 检查3005端口（admin-panel）
netstat -ano | findstr :3005

# 检查3003端口（server）
netstat -ano | findstr :3003

# 终止占用端口的进程（替换PID为实际值）
taskkill /F /PID <占用端口的进程ID>
```

### 步骤2：清理缓存和依赖
```bash
# 清理npm缓存
npm cache clean --force

# 重新安装依赖（在项目根目录）
npm install

# 分别进入各目录安装依赖
cd apps/server && npm install
cd ../h5-client && npm install
cd ../admin-panel && npm install
cd ../..
```

### 步骤3：重启开发服务器
```bash
# 启动server端（在server目录）
cd apps/server && npm run start:dev

# 启动admin-panel端（在admin-panel目录）
cd ../admin-panel && npm run dev

# 启动h5-client端（在h5-client目录）
cd ../h5-client && npm run dev
```

### 步骤4：验证服务启动状态
- 访问 server：http://localhost:3003/api
- 访问 admin-panel：http://localhost:3005
- 访问 h5-client：http://localhost:5174

### 步骤5：处理可能的其他问题
- **如果遇到权限问题**：以管理员身份运行终端
- **如果遇到依赖冲突**：删除node_modules目录后重新安装
- **如果端口仍然被占用**：修改配置文件中的端口号

## 预期结果

- ✅ 无端口冲突错误
- ✅ 开发环境恢复正常
