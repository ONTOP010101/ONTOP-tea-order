# Git Push操作计划（包含分享订单显示修改）

## 1. 检查Git状态
- 执行 `git status` 命令，确认所有修改的文件
- 验证分享订单相关文件是否已包含在修改中

## 2. 添加修改文件
- 执行 `git add .` 命令，将所有修改和新增的文件添加到暂存区
- 确保以下关键文件都已添加：
  - `apps/h5-client/src/views/OrderList.vue` - 分享订单弹窗布局优化
  - `apps/h5-client/src/views/OrderDetail.vue` - 分享订单弹窗布局优化
  - `apps/h5-client/index.html` - 资源预加载和预取
  - `apps/h5-client/vite.config.ts` - 代码分割和构建配置优化
  - `apps/h5-client/src/router/index.ts` - 路由预加载配置
  - `apps/h5-client/src/utils/request.ts` - API缓存策略
  - `apps/h5-client/src/utils/performance.ts` - 性能监控工具
  - `apps/h5-client/src/main.ts` - 性能监控初始化
  - `apps/h5-client/package.json` - 添加web-vitals依赖
  - `apps/server/src/modules/order/order.service.ts` - 订单服务修复

## 3. 提交修改
- 执行详细的提交信息，包含所有修改内容：
```bash
git commit -m "优化前端功能和性能：
- 优化分享订单弹窗显示：
  - 改为grid布局，显示2-3道菜/行
  - 优化移动设备显示，尽量一屏显示所有菜品
  - 添加菜品图片、名称、规格和数量显示
  - 美化弹窗样式，添加阴影和边框
- 优化前端页面加载速度：
  - 添加资源预加载和预取策略
  - 优化代码分割和构建配置
  - 实现页面切换性能优化（路由预加载）
  - 添加API缓存策略
  - 集成性能监控工具
  - 运行时优化（防抖、懒加载）
- 修复服务器端订单服务：
  - 修复clearExpiredOrders方法语法错误"
```

## 4. Push到远程仓库
- 执行 `git push origin master` 命令，将修改推送到远程仓库
- 验证push操作是否成功

## 5. 验证推送结果
- 检查远程仓库是否已更新
- 确认所有修改文件都已正确推送

## 分享订单显示修改要点
- **布局优化**：使用grid布局实现多列显示
- **移动适配**：紧凑布局，适合手机屏幕
- **信息完整**：显示菜品图片、名称、规格和数量
- **视觉美化**：添加阴影、边框和适当的间距
- **响应式设计**：自动调整列数，适应不同屏幕尺寸

## 操作目标
确保所有修改（包括分享订单显示和前端性能优化）都能成功推送到远程仓库，为后续部署做准备。
