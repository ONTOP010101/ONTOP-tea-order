# 修复搜索商品失败的问题

## 问题分析
- **错误信息**：`搜索商品失败: AxiosError: Request failed with status code 400`
- **根本原因**：前端代码调用了 `/products/search` 端点，但后端没有实现这个路由
- **现状**：
  - 前端在 `h5-client/src/api/product.ts` 中定义了 `searchProducts` 函数，调用 `/products/search` 端点
  - 后端在 `server/src/modules/product/product.controller.ts` 中没有实现 `/products/search` 路由
  - 后端已经在 `/products` 端点中支持了搜索功能，通过 `keyword` 参数实现

## 解决方案
修改前端的 `searchProducts` 函数，使其使用现有的 `/products` 端点，而不是不存在的 `/products/search` 端点。

## 具体修改

### 修改前端搜索函数
**文件**：`c:\Users\wd439\Documents\trae_projects\ONTOP\apps\h5-client\src\api\product.ts`

**原代码**：
```typescript
// 搜索商品
export const searchProducts = (keyword: string, params?: PaginationParams) => {
  return request.get<any, PaginationResponse<Product>>('/products/search', { 
    params: { keyword, ...params } 
  })
}
```

**修改后**：
```typescript
// 搜索商品
export const searchProducts = (keyword: string, params?: PaginationParams) => {
  return request.get<any, PaginationResponse<Product>>('/products', { 
    params: { keyword, ...params } 
  })
}
```

## 预期效果
- 前端搜索商品时不再调用不存在的 `/products/search` 端点
- 前端使用现有的 `/products` 端点进行搜索
- 后端能够正确处理搜索请求，返回搜索结果
- 搜索功能恢复正常，不再出现 400 错误
