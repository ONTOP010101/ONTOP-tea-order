### 找到Node.js的实际安装路径

根据您提供的截图，系统中确实安装了Node.js 18.18.2，但命令行无法识别`node`命令，这可能是因为安装路径未添加到系统的PATH环境变量中。

请按照以下步骤操作：

1. **打开Node.js安装路径**：
   - 右键点击开始菜单中的Node.js图标
   - 选择"打开文件位置"
   - 这将打开包含Node.js快捷方式的文件夹
   - 再次右键点击Node.js快捷方式，选择"打开文件位置"
   - 这将显示Node.js的实际安装路径（例如：`C:\Program Files\nodejs`）

2. **验证Node.js是否能正常运行**：
   - 打开命令提示符（CMD）
   - 导航到Node.js安装路径（例如：`cd "C:\Program Files\nodejs"`）
   - 运行 `node -v` 命令，应该显示 `v18.18.2`
   - 运行 `npm -v` 命令，应该显示对应的npm版本

3. **将Node.js添加到PATH环境变量**：
   - 按Win+R键，输入`sysdm.cpl`，点击确定
   - 点击"高级"选项卡，然后点击"环境变量"
   - 在"系统变量"中找到"Path"变量，点击"编辑"
   - 点击"新建"，添加Node.js的实际安装路径
   - 点击"确定"保存所有更改
   - 重启命令提示符，再次运行 `node -v` 命令

4. **启动H5客户端**：
   - 导航到H5客户端目录：`cd c:/Users/wd439/Documents/trae_projects/ONTOP/apps/h5-client`
   - 运行 `npm run dev` 命令，启动开发服务器
   - 访问 http://localhost:5174

### 预期结果

- `node -v` 和 `npm -v` 命令能正常显示版本信息
- `npm run dev` 命令能成功启动H5客户端
- 前端能正常访问后端API

### 注意事项

- 确保在修改环境变量后重启命令提示符
- 如果仍然无法识别node命令，可能需要重启计算机
- 如果安装路径包含空格，请使用引号包裹路径

按照以上步骤操作后，您应该能够成功启动H5客户端。如果还有问题，请随时告知！
